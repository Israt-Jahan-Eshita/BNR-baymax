# Dashboard Aggregate API

The Dashboard Aggregate API provides a unified view of the super-agent's state across multiple bounded domains.

## Purpose
Instead of requiring the frontend client to make 6 separate HTTP requests on initial load (Balances, Forecasts, Data Health, Alerts, Cases, Scenario Runs), the frontend can issue one request to `/api/v1/dashboard/aggregate`.

## Data Included
- **Balances**: Real-time physical cash and e-money balances across providers.
- **Liquidity Forecast**: Predicts runway based on recent historical consumption rate.
- **Data Health**: Indicates the confidence and status (LIVE, DELAYED, MISSING, CONFLICTING) of the data feeding the balance model.
- **Recent Alerts**: The 5 most recent actionable alerts generated by the anomaly engine.
- **Recent Cases**: The 5 most recent operational cases derived from critical alerts.
- **Recent Scenario Runs**: The 5 most recent runs from the Scenario Engine testing framework.

This single unified payload allows the frontend to rapidly paint the UI for the "See-Predict-Inject-Detect-Act-Validate" dashboard.
